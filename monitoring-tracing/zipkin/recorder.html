Recorder

Recorder 主要用于在本地保存 Span，并执行对 Span 中数据的操作，直到 Span 被发送给 Zipkin。

Recorder

Recorder 是一个实现类，它拥有 4 个字段：

final MutableSpanMap spanMap;
final Clock clock;
final Reporter<zipkin2.Span> reporter;
final AtomicBoolean noop;
  • noop 用来标记是否真的需要执行实际的数据操作。
  • clock 用来获取平台相关的时间信息。
  • spanMap 用来保存实际的 Span 数据。
  • reporter 用来向 Zipkin 发送 Span。

除了在结束 Span 时需要使用 reporter 向 Zipkin 发送数据，其他操作在非 noop 的情况下均围绕对 spanMap 的操作展开。

MutableSpanMap

MutableSpanMap 内部使用一个 ConcurrentMap<Object, MutableSpan> 来保存具体的 Span 数据，奇妙的是这个 Key 的类型为 Object，因为在存数据时使用 RealKey，而查找数据时使用 LookupKey，下面是这两个键类型的声明：

static final class RealKey extends WeakReference<TraceContext> {
	RealKey(TraceContext context, ReferenceQueue<TraceContext> queue) {
      	super(context, queue);
      	hashCode = context.hashCode();
    }
    // omit..
}

tatic final class LookupKey {
	LookupKey(TraceContext context) {
      	this.context = context;
    }
    
    @Override public int hashCode() {
      	return context.hashCode();
    }
    // omit..
}

TraceContext 的 hashCode 是 final 的，因此 RealKey 和 LookupKey 均以其 hashCode 作为自身的 hashCode，以实现在 Map 中对同一 TraceContext 的存取。

而 RealKey 实现为一个 WeakReference，使用弱引用作为 Map 的键可以利用 GC 将没有被引用的对象移除。MutableSpanMap 本身继承了 ReferenceQueue，当这些 RealKey 关联的 Span 被 GC 移除时会被放置到 ReferenceQueue 的队列中。这时，MutableSpanMap 再在每次存取数据时顺便调用 reportOrphanedSpans 读取该队列中被 GC 的 Span，执行最后的操作，将这些 Span 发送给 Zipkin。这样避免忘记被关闭的 Span 在内存中堆积，并且不会丢失任何 Span：

void reportOrphanedSpans() {
	Reference<? extends TraceContext> reference;
	while ((reference = poll()) != null) {
      	TraceContext context = reference.get();
      	MutableSpan value = delegate.remove(reference);
      	if (value == null || noop.get()) continue;
      	try {
        	value.annotate(value.clock.currentTimeMicroseconds(), "brave.flush");
        	reporter.report(value.toSpan());
      	} catch (RuntimeException e) {
        	// don't crash the caller if there was a problem reporting an unrelated span.
        	if (context != null && logger.isLoggable(Level.FINE)) {
          		logger.log(Level.FINE, "error flushing " + context, e);
        	}
		}
	}
}

这便是一个典型的、利用 WeakReference 和 ReferenceQueue 实现的弱引用缓存。

另外，其 getOrCreate 方法会基于 TraceContext 尝试查找已存在的 Span，如果不存在，在创建一个新的 Span 并返回：

MutableSpan getOrCreate(TraceContext context) {
    MutableSpan result = get(context);
    if (result != null) return result;

    // save overhead calculating time if the parent is in-progress (usually is)
    Clock clock = maybeClockFromParent(context);
    if (clock == null) {
      clock = new TickClock(this.clock.currentTimeMicroseconds(), System.nanoTime());
    }

    MutableSpan newSpan = new MutableSpan(clock, context, endpoint);
    MutableSpan previousSpan = delegate.putIfAbsent(new RealKey(context, this), newSpan);
    if (previousSpan != null) return previousSpan; // lost race
    return newSpan;
}

而 remove 方法则会将对应 TraceContext 的 Span 移除：

@Nullable MutableSpan remove(TraceContext context) {
    if (context == null) throw new NullPointerException("context == null");
    MutableSpan result = delegate.remove(new LookupKey(context));
    // also clears the reference relating to the recent remove
    reportOrphanedSpans();
    return result;
}

MutableSpan

MutableSpanMap 的值类型 MutableSpan 是一个包可见的类，主要是对 Span 数据操作方法的封装。
