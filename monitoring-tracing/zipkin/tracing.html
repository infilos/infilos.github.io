Tracing

Brave 是兼容 Zipkin 协议的客户端实现，用于以代码形式创建追踪信息(即 Span)。

Tracing

Tracing 类用于设置最基本的信息，并用于创建或获取追踪操作入口 Tracer 类的实例。Traceing 是一个抽象类，并在内部提供了一个 Defualt 默认实现以及对应的 Builder。因此我们可以直接阅读其 Builder 实现来观察其构建过程。

public static final class Builder {
    String localServiceName;
    Endpoint endpoint;
    Reporter<zipkin2.Span> reporter;
    Clock clock;
    Sampler sampler = Sampler.ALWAYS_SAMPLE;
    CurrentTraceContext currentTraceContext = CurrentTraceContext.Default.inheritable();
    boolean traceId128Bit = false;
    boolean supportsJoin = true;
    Propagation.Factory propagationFactory = B3Propagation.FACTORY;
    ErrorParser errorParser = new ErrorParser();
 	
    // omitted....
    
	public Tracing build() {
      	if (clock == null) clock = Platform.get().clock();
      	if (endpoint == null) {
        	endpoint = Platform.get().endpoint();
        	if (localServiceName != null) {
          	endpoint = endpoint.toBuilder().serviceName(localServiceName).build();
        	}
      	}
      	if (reporter == null) reporter = Platform.get().reporter();
      	return new Default(this);
    }
}

Builder 中的核心属性包括：

  • localServiceName，当前服务的名称。
  • endpoint，当前服务的端点信息，包括：serviceName、ipv4、ipv6、port。
  • reporter，用于向 Zipkin 发送追踪信息(Span)，在一个独立的仓库中实现。
  • clock，用于获取当前微秒单位时间的时钟，Zipkin 中所有时间的单位均为微秒。
  • sampler，采样器，包含一组策略来决定当前 Traceid 是否需要被采样(即需要被发送)。
  • currentTraceContext，用于获取当前 TraceContext 的辅助方法，用于在进程内传播追踪信息。
  • traceId128Bit，Traceid 是否采用 128 位。
  • supportsJoin，是否支持 Span 的连接，如果支持，则客户端与服务端将共用一个 Spanid，即不是在服务端创建一个子 Span。
  • propagationFactory，传播器工厂，用于创建传播器实例，传播器决定了在跨进程传播追踪信息时如何向载体(Carrier)中注入追踪信息、如何从载体中提取追踪信息。比如以 HTTP 载体，客户端将追踪信息添加到 HTTP 首部，服务端再从 HTTP 首部中提取追踪信息。
  • errorParser，用于格式化异常信息，比如将异常的 message 添加到追踪信息中，并为追踪信息添加一个 Error 的标签。

调用 build 方法内部将会创建一个 Tracing.Defautl 实例并返回：

static final class Default extends Tracing {
    final Tracer tracer;
    final Propagation.Factory propagationFactory;
    final Propagation<String> stringPropagation;
    final CurrentTraceContext currentTraceContext;
    final Sampler sampler;
    final Clock clock;
    final ErrorParser errorParser;

    Default(Builder builder) {
      this.clock = builder.clock;
      this.errorParser = builder.errorParser;
      this.propagationFactory = builder.propagationFactory;
      this.stringPropagation = builder.propagationFactory.create(Propagation.KeyFactory.STRING);
      this.currentTraceContext = builder.currentTraceContext;
      this.sampler = builder.sampler;
      this.tracer = new Tracer(
          builder.clock,
          builder.propagationFactory,
          builder.reporter,
          new Recorder(builder.endpoint, clock, builder.reporter, noop),
          builder.sampler,
          builder.errorParser,
          builder.currentTraceContext,
          builder.traceId128Bit || propagationFactory.requires128BitTraceId(),
          builder.supportsJoin && propagationFactory.supportsJoin(),
          noop
      );
      maybeSetCurrent();
    }
    // omitted....
}

在创建 Tracing.Default 实例的同时，会在其构造器内部创建一个 Tracer 实例并赋值给 Tracing 的 tracer 字段，后续则可以通过 Tracing 的静态方法 currentTracer 来获得当前的 Tracer 实例。
